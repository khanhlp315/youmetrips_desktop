028   * A person's public profile refers to the following properties on the user object by default:

030   * <li>id</li>
031   * <li>name</li>
032   * <li>first_name</li>
033   * <li>last_name</li>
034   * <li>link</li>
035   * <li>gender</li>
036   * <li>locale</li>
037   * <li>timezone</li>
038   * <li>updated_time</li>
039   * <li>verified</li>
040   * </ul>
041   * 
042   * On the web, public_profile is implied with every request and isn't required, although the best practice is to
043   * declare it. On iOS and Android, you must manually request it as part of your login flow.<br />
044   * <br />
045   * 
046   * gender & locale can only be accessed if:
047   * 
048   * <ul>
049   * <li>The person queried is the person using the app.</li>
050   * <li>The person queried is using the app, and is a friend of the person using the app.</li>
051   * <li>The person queried is using the app, is not a friend of the person using the app, but the app includes either
052   * an app access token or an appsecret_proof argument with the call.</li> <br />
